# アクセス権限 {#access-rights}

Adobe ID の概要と、それを使用して Experience Cloud リソースにアクセスする方法について説明します。

## ユーザーの ID タイプ {#user-identity-types}

オンボーディングプロセスの一環として、Adobeでは、Adobe Identity Management システム（IMS）で会社用の組織IDを作成します。 IMS では、すべてのユーザーとその権限が管理されます。 この組織のメンバーである必要があり、[!UICONTROL Experience Cloud] サービスへのアクセス権が付与されている各ユーザーには、Adobe IDが必要です。

Adobe IDについて詳しくは、[Adobe ID タイプの管理](https://helpx.adobe.com/business/enterprise/set-up-your-organization/identity.html)を参照して、使用可能なID タイプのいずれかを使用してAdobe IDを取得する方法について詳しく説明してください。

### ユーザーと役割 {#users-and-roles}

アドビがお客様の会社の組織を作成すると、指定した管理者がこの組織の最初のメンバーとして追加されます。 デフォルトでは、システム管理者は管理者権限を受け取ります。 システム管理者には、[!UICONTROL AEM Managed Services]製品と1つ以上の[!UICONTROL Cloud Manager]製品プロファイルも割り当てられます。 Admin Consoleを使用してチームを設定および管理する方法については、[Add Users and Roles](/help/requirements/users-and-roles.md)を参照してください。

これらの権限が付与されると、管理者はAdobe IDを使用してシングルサインオン用に設定され、[!UICONTROL Experience Cloud] サービスにアクセスし、AEM クラウド環境にログインして、[!UICONTROL Cloud Manager]を使用できるようになります。
