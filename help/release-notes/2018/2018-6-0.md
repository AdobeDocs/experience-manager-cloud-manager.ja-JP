# リリースノート（2018.6.0） {#release-notes-for}

次の節では、[!UICONTROL Cloud Manager] 2018.6.0の一般的なリリースノートの概要を説明し、デプロイ時のDispatcher無効化、通知の追加、およびユーザビリティの向上のサポートを追加します。

## リリース日 {#release-date}

[!UICONTROL Cloud Manager] 2018.6.0 のリリース日は 2018年8月9日（PT）です。

## 新機能 {#what-s-new}

* **CI/CD パイプライン**

   * CI/CD （Continuous Integration/Continuous Delivery）パイプラインの実行中に、ステージと実稼動環境の両方で設定可能なDispatcherの無効化とキャッシュフラッシュを実行します。 詳しくは、[実稼動パイプラインの設定](/help/using/production-pipelines.md)を参照してください
   * パイプラインの設定中に、品質ゲートの1つで重要なエラーが発生した場合にパイプラインがどのように動作するかを定義できるようになりました。 詳しくは、[実稼動パイプラインの設定](/help/using/production-pipelines.md)を参照してください。
   * パイプラインのセットアップ中に、CSE （カスタマーサクセスエンジニア）または利用可能なCSEがCSE監視を実行するかどうかを選択できるようになりました。 詳しくは、[実稼動パイプラインの設定](/help/using/production-pipelines.md)を参照してください。
   * CI/CD パイプラインがビジネス承認ステップに到達すると、デプロイメントの承認を許可されたユーザーに通知が送信されます。 詳しくは、[通知](/help/using/notifications.md)を参照してください。
   * CI/CD パイプラインがスケジュールセットに到達すると、スケジュールを設定する権限を持つユーザーに通知が送信されます。 詳しくは、[通知](/help/using/notifications.md)を参照してください。
   * 誤って設定されたアウトバウンド HTTP/HTTPS リクエストを識別する新しいルール。 詳しくは、[ カスタムコード品質ルール ](/help/using/custom-code-quality-rules.md)を参照してください。

## バグ修正 {#bug-fixes}

* 場合によっては、パフォーマンステストがDispatcherとパブリッシャーの複数のインスタンスに完全に分散されていませんでした。
* 狭いブラウザーウィンドウでは、ヘッダーナビゲーションが非表示になっていました。
* パイプラインの実行中に一部のパイプライン設定が無効になりませんでした。
* プログラムリスト画面のAEMとモニタリングへのリンクは、現在のブラウザータブを置き換え、新しいタブを開きました。
* 状況によっては、承認期間が長く続くとデプロイメントに失敗することがありました。
